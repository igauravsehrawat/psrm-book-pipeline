Huang's termination detection algorithm                         

  • The algorithm can be described as:
  • Initially all processes are idle.
  • A distributed task is started by a process sending a computational message to another process. This initial process to send the message is the "controlling agent".
  • The initial weight of the controlling agent is  (usually 1).
  • The following rules are applied throughout the computation:
  • A process sending a message splits its current weight between itself and the message.
  • A process receiving a message adds the weight of the message to itself.
  • Upon becoming idle, a process sends a message containing its entire weight back to the controlling agent and it goes idle.
  • Termination occurs when the controlling agent has a weight of  and is in the idle state.

Some weaknesses to Huang's algorithm are that it is unable to detect termination if a message is lost in transit or if a process fails while in an active state.

                                                             Ideology

  • The algorithm was proposed by Shing-Tsaan Huang in 1989 in the Journals of Computers.
  • One of the earliest termination detection algorithms, which was proposed by Dijkstra and Scholten [2], assumes that the underlying computation is diffusing(starts from a single source).Huang’s weight throwing algorithm is based on a similar idea.
  • The weights are infinitely divisible.
  • The algorithm was generated with the idea of the deadlock detection using graph search technique in the system.
  • In searching a graph with logical edges, the searching can start from a root with a weight of one. The root sends a token T(DW := 1/ I NSI) to each of its neighbors. We use NS to denote the set of the neighbors of a node. Upon receiving a token T(DW) for the first time, a node sends a token T(DW := DW/(INSI-l)) to each of its neighbors except the one from which the token is received. For a revisited token T(DW), a node sends a T(DW) to the root. A sink node, a node with only one neighbor, sends a T(DW) to the root upon receiving a token . The root detects the termination of the searching when the sum of the weights on all the returned tokens equals one. This search technique assumes that a weight is infinitely divisible
  • Due to absence of global clock in distributed system ,there was a need of some fixed  local parameter to detect the termination. So weights are used to detect the termination.
  • Lets say there are 2 processes in a distributed system.the tasks is distributed between them ,i.e, half of the task to each or the task can be divided in any ratio between the two .To detect termination I need to check if each has completed their task or might have crashed.for that I need the completed task to be returned and summed up which is equal to  the the big task. Now to sum up my subtasks I also need an external agent as there is no global clock in the Distributed system.
  • Now let’s assume there are 3  processes in a distributed system. the tasks is distributed between them as they are in a distributed system ,i.e, one third of the task to each or the task can be divided in any ratio between the two .To detect termination I need to check if each has completed their task or might have crashed.For that I need the completed task to be returned and summed up which is equal to  the the big task. Now to sum up my subtask I also need an external agent.

 

  • Now let’s assume there are 6 processes in a distributed system. So lets say that initially the big task to be completed is equal to x.

Now if there are 6 processes ,the task is distributed among them in some ratio.

So lets say x/2 to 1st process. now x/2 of the task is remaining.now the 2nd process comes in and is ready for the task so x/4 is given to it. Remaining part of work=x/4.

Now 3rd process is active and gets x/8. 4th ,5th and 6th gets x/16,x/32,x/64 .Now the external agent is left with x/64.

So to see if my task is completed ,I need to check the sum of all the subtask to be equal to x ,i.e,

x=x/2+x/4+x/8+x/16+x/32+x/64+x/64

the external agent is always left with some work as it is always active and doing work.

  • Now let’s assume there are N processes. so every time the process is active (readily available for a task) ,the external agent gives half of its task to the active process. To make it easy let the external agent task be equal to one and assume that the weight is infinitely divisible.

In general,

1=1/2+1/2^2+1/2^3+1/2^4+1/2^5…………………………..1/2^N+1/2^N.

 

Once we the sum is equal to one ,the termination is detected as all the processes have completed their task and the task has been integrated together.

 

Idea behind Bellman Ford algorithm:

  • It first calculates the shortest distances for the shortest paths which have at-most one edge in the path. Then, it calculates shortest paths with at-nost 2 edges, and so on.
  • After the ith iteration of outer loop, the shortest paths with at most i edges are calculated. There can be maximum |V| – 1 edges in any simple path, that is why the outer loop runs |v| – 1 times.\
  • The idea is, assuming that there is no negative weight cycle, if we have calculated shortest paths with at most i edges, then an iteration over all edges guarantees to give shortest path with at-most (i+1) edges.

 

Token Ring algorithm

 

  • It is an algorithm to implement mutual exclusion in which each process must request permission to enter its critical section and must release it after it has completed its execution. A mutual exclusion algorithm must satisfy the following requirements [1, 2]:

i.    At most one process can execute its critical section at a given time.

ii.   If no process is in its critical section, any process requesting to enter its critical  section must be allowed to do so at finite time.

iii. When competing processes concurrently request to enter their respective critical sections, the selection cannot be postponed indefinitely.

iv.  A requesting process cannot be prevented by another one to enter its critical section within a finite delay.

  • In this algorithm, the nodes in the system form a logical ring. A token is passed around the ring. A node can enter the critical section if it holds the token.The competing nodes generate a token for the permission to enter CS. The token traverses the logical ring structure. A node can enter CS if and only if it receives back its generated token. In average N/ 2 messages are required to handle one request in a N node system.

                                                                       Ideology

  • To ensure Mutual exclusion, we have to ensure that no two processes accesses the critical section at the same time. So by using one token we can ensure that the process having the token can enter the critical section and other processes have to wait till they get the token. But to maintain fairness and remove starvation, the order of processes should be such that each process gets a fair chance to enter the critical section in a reasonable amount of time. So we assume a ring topology where the token is passed to all the processes one by one and each process gets a fair chance to enter the critical section.

 

  • Suppose there are 3 processes in the distributed system and initially the token is with process1, so it can access the critical section. If process2 wants to enter the critical section it will send the request to process1, and process1 will send the token to process2 if the critical section is free.
  • Similarly each process can access the critical section and no two processes can enter it at the same time as there is only 1 token in the distributed system.
