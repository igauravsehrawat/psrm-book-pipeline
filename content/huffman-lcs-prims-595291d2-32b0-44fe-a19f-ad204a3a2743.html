HUFFMAN ENCODING

The problem statement in front of the inventor:

To find an efficient method of representing numbers, letters or other symbols using a binary code, in such a way that the average number of coding digits per message is minimized. Also, it should give the encoded message that can be decoded easily and uniquely.

Procedure:

  • To encode n different symbols, we need log(n) bits to encode, if we use fixed length coding. Therefore, to reduce the average number of bits to even below log(n), variable length coding is essential, provided that it fulfills the basic requirement of generation of a unique code for all the characters.

Conclusion: Variable length coding is necessary.

 

  • To assign variable length codes to different symbols, there has to be parameter that distinguishes the symbols. The most suitable parameter can be the probability of occurrence of a particular symbol in the entire message.

Conclusion: Criteria for evaluating symbols will be probability of occurrence.

 

  • Since the criteria for evaluating different symbols were there probability, the inventor might have inclined to use frequency of the symbols in a message to create code words. Therefore, the symbol with higher probability will get lesser number of bits in code and symbol with less probability will get longer code.

Conclusion: Use frequency to calculate probability of occurrence.

 

  • As the binary code finally produced will be a mere string of 0s and 1s without any delimiter to identify where the next symbol begins, a way to identify different symbols has to be ensured. Since the encoding being used is variable length encoding, therefore fixed bits for each symbol can’t be used. So there has to be a pattern in the encoding that ensures detection of each symbol separately at the same time ensuring the results are always unique. One such way of identifying patterns is prefix matching, i.e. ensuring a type of coding system, which states that there is no valid code word in the system that is a prefix of any other valid code word in the set.

Conclusion: Prefix matching to be used.

 

  • Code words with prefix coding can be obtained using a tree data structure. As there is only one unique path from the root to each leaf, thereby ensuring a prefix code for each code word. The leaf nodes of the tree will denote a symbol, and the path from root to the leaf will uniquely determine which symbol it is.

Conclusion: Use of tree.

 

  • As only binary strings characters can be used, the inventor might have made a direct leap from a tree data structure to binary tree data structure.

Conclusion: Use of binary tree.

 

  • Now, to reduce the average number of bits, the best way, inventor might have thought, would be to place symbols with largest frequencies closest to root. This, in a way, ensures the principle of variable length coding, i.e. symbols with largest frequencies be represented by smallest code words and vice-versa, to minimize average number of bits.

Conclusion: The final tree should have distance of each symbol from the root inversely proportional to its probability of occurrence.

 

  • To build the tree, the inventor might have tried to build the tree from top to bottom i.e. starting from highest probability to lowest, as well as, from bottom to top i.e. from lowest to highest probability. By comparison, he must have concluded that the later can give an optimal solution every time. The comparison is illustrated below.

Conclusion: Build tree bottom-up.

 

  • To generate unique code, the tree’s branches should determine whether the next code bit is 0 or 1, therefore, a pattern such as left child given 0 code and right child given 1 code has to chosen.

Conclusion: The branch location (left/right) will determine the next code bit to be ‘0’ or ‘1’ in that iteration.

PRIM'S ALGORITHM


The problem statement in front of the inventor:

To derive a minimum spanning tree (MST) from a graph G, where G is an undirected, connected graph such that the number of vertices is V, and the number of edges is E.

Prescript: Prim’s algorithm was introduced to find the electric circuitry between houses in a manner that uses minimal material (thereby most economical). It can also be used for maze generation.

 

  • Since in an MST, every node has to be traversed, the inventor must have thought that a randomized arbitrary choice of starting vertex is better than computing some formula for such selection as it prevents additional computational overhead, and in some cases, even worst-case scenarios.

Conclusion: Random first node

 

  • Choosing the edge with the minimum weight (from the starting vertex) is the obvious next step. Thus, the first two vertices are selected.

 

  • By comparison, the inventor must have noted that there are two ways to select the third, fourth.. nth vertex that can be followed:

- Selecting the minimum edge from the second vertex; then selecting the minimum edge from the third vertex; and so on. This will create a tree with exactly one node at every level. This is impractical in real life. And space inefficient, as well.

- Selecting the minimum edge from the subset of currently selected vertices. This creates a m-tree with a collection of all minimum edges, as such whose sum is the minimum. This is more useful for real purposes.

 

  • Now, we have an m-tree containing each and all minimum edges. Proof (by contradiction) that this is, in fact, the minimum spanning tree:

Let us assume that T be an optimal minimum spanning tree of the undirected, connected graph G, with edges E and vertices V.

Suppose, on the contrary, there exists a minimum edge between vertices u and v, ev that is not part of T.

To make it an MST, we must now add ev to T.

However, that creates a cycle in the minimum spanning tree T. [u, v, w, … , u]

As such, we now have to delete either ev or the edge between vertices v and w.

But, it has already been given that weight of ev is lesser than that of weight of the edge between v and w. If, however, we remove this edge to include ev instead, it would lead to a minimum spanning tree cheaper than T

Contradiction! Hence proved by contradiction. Thus, the tree we deduced is in fact the minimum spanning tree.

LONGEST COMMON SUBSEQUENCE

The problem statement in front of the inventor:

Given two strings: string S of length n, and string T of length m. Produce their longest common subsequence: the longest sequence of characters that appear left-to-right (but not necessarily in a contiguous block) in both strings.

 Naive approach: A string of length n has 2n different possible subsequences. The naive solution for this problem is to generate all subsequences of both given sequences and find the longest matching subsequence. So, the time complexity would be of the order O(2(m+n)), where m and n are the lengths of the two strings.

Dynamic programming approach: The inventor must first realize that the LCS problem can be solved using dynamic programming. If a problem exhibits the properties of optimal substructure and overlapping substructure, then the concepts of dynamic programming can be used to solve said problem.

A problem is said to have optimal substructure if an optimal solution can be constructed efficiently from optimal solutions of its subproblems.

A problem is said to have overlapping substructure if the problem can be broken down into subproblems which are reused several times or a recursive algorithm for the problem solves the same subproblem over and over rather than always generating new subproblems.

Let the subproblems consists of LCS of a prefix of S and a prefix of T, running over all pairs of prefixes.

Proof for optimal substructure property:

So, the problem becomes this: say LCS[i,j] is the length of the LCS of S[1..i] with T[1..j]. How can one solve for LCS[i,j] in terms of the LCS’s of the smaller problems?

Case 1: what if S[i] ≠ T[j]? Then, the desired subsequence has to ignore one of S[i] or T[j] so we have:

LCS[i, j] = max(LCS[i − 1, j], LCS[i, j − 1]).

Case 2: what if S[i] = T[j]? Then the LCS of S[1..i] and T[1..j] might as well match them up. For instance, if I gave you a common subsequence that matched S[i] to an earlier location in T, for instance, you could always match it to T[j] instead. So, in this case we have:

LCS[i, j] = 1 + LCS[i − 1, j − 1].

Examples:

1) Consider the input strings “AGGTAB” and “GXTXAYB”. Last characters match for the strings. So length of LCS can be written as:

L(“AGGTAB”, “GXTXAYB”) = 1 + L(“AGGTA”, “GXTXAY”)

2) Consider the input strings “ABCDGH” and “AEDFHR. Last characters do not match for the strings. So length of LCS can be written as:

L(“ABCDGH”, “AEDFHR”) = MAX ( L(“ABCDG”, “AEDFHR”), L(“ABCDGH”, “AEDFH”) )

 

So the LCS problem has optimal substructure property as the main problem can be solved using solutions to subproblems.

    

Proof for overlapping subproblems property:

Consider the example of two input strings “AXYT” and “AYZX”:

In the above partial recursion tree, LCS(“AXY”, “AYZ”) is being solved twice. If one draws the complete recursion tree, then he can see that there are many subproblems which are solved again and again. So this problem has overlapping substructure property and re-computation of same subproblems can be avoided by either using Memoization or Tabulation.

So the LCS problem has overlapping substructure property, as well, as the solutions to some subproblems can be repeatedly used to solve the main problem.

Thus, LCS can be solved using dynamic programming.

Using dynamic programming, the inventor can just do two loops (over values of i and j), filling in the LCS using the rules as described by the two cases. A pictorial representation, with S along the leftmost column and T along the top row is shown below.

Let,

T = BACBAD


Now, fill out this matrix row by row, doing constant amount of work per entry, so this takes O(mn) time overall. The length of the LCS of S and T is in the lower-right corner.

To find the sequence, just walk backwards through matrix starting the lower-right corner. If either the cell directly above or directly to the right contains a value equal to the value in the current cell, then move to that cell (if both to, then chose either one). If both such cells have values strictly less than the value in the current cell, then move diagonally up-left (this corresponds to applying Case 2), and output the associated character. This will output the characters in the LCS in reverse order. For instance, running on the matrix above, this outputs DABA.
