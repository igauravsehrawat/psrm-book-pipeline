SEARCHING ALGORITHMS

Search algorithm is an algorithm for finding an item with specified properties among a collection of items. The items may be stored individually as records in a database; or may be elements of a search space defined by a mathematical formula or procedure, such as the roots of an equation with integer variables; or a combination of the two.

 

THOUGHT PROCESS

To search for items in a given space is a very common problem for humans since time immemorial. So to achieve this task  humans used to follow various techniques. After gathering all the data human tendency would be to begin search from that data set where the probability of finding the the desired element is   maximum. This primary step of each technique would be to gather information or data from all the potential sources. After gathering all the data human tendency would be to begin search from that data set where probability of finding the element is maximum .To create  the group with maximum probability various techniques could be used. These techniques are developed using some property of the data present.One technique would be to arrange element in some order and then dividing the whole list into subset,and selecting the proper subset .So this technique is used by binary search algorithm. 

BINARY SEARCH

If the list to be searched contains few items, then it would be easy to search a particular element in it but if the list is large then it will be tedious task. Thus to reduce the search space subsets are to formed using some property which could segregate the elements. One simple technique to form subset is to arrange elements in some order and then forming subsets using some conditions. And this technique is used in binary search. Here the list is sorted and  divided into halves depending on condition and then a proper subset is identified in which the element is found. We do comparisons and find out the subset in which the element is present. By this way eliminate rest of the array. Hence our search space is reduced to half in each iteration, hence the cost and time to search the element is also reduced. To achieve this we compare the element the element to be searched with the mid element and after comparison the proper subset is identified.

To solve this problem one approach is to assume that sequence S is ordered( for instance S is monotonically increasing sequence) and then to proceed as follows:

  • Compare x with S[K], (1<K<=T)
  • If x=S[K], then the search is successful and the meassge ‘x έ S’ is  generated;
  • If x<S[K], then the search proceeds within the sequence S[1:K-1];
  • If x>S[K], then the search proceeds within the sequence S[K+1:T];
  • If the search sequence is empty, the search is unsuccessful and the message ‘x does not belongs to S’ is generated.

Assuming that each of the T terms is equally likely to be the number sought for, it is clear that an efficient search method will have the search sequence on each iteration step. Thus a search requires about log2T comparisons.

 

Some of the very common every day application of binary search is finding words in a dictionary or finding phone numbers from a phone directory.
