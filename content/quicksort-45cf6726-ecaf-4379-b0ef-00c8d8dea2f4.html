Concept

• Divide and Conquer paradigm may ease solution by trivializing it because it is a powerful tool for solving conceptually difficult problems: all it requires is a way of breaking the problem into subproblems, of solving the trivial cases and of combining sub-problems to the original problem (Sorting two smaller array is more time efficient than sorting one huge array).

• After that comes the idea how to divide the array into sub arrays efficiently.

• The idea of dividing the array may be developed by using a property of binary search tree that the left subtree of a node contains only nodes with keys less than the node's key and the right subtree of a node contains only nodes with keys greater than the node's key.



• From this the concept pivot element may come which will further help to divide the array into sub arrays.

• Similarly, a pivot (an element from the array) must be selected from the array which may divide the array same as BST that the left side of the pivot must contains the elements less than it and the right side contains the elements more than the pivot.

• Now there should be a way of selecting the pivot element because suppose instead our pivot element splits the array as unequally as possible. Thus instead of elements in the smaller half, we get zero, meaning that the pivot element is the biggest or smallest element in the array. Thus the worst case time for this algorithm is worse than other divide and conquer algorithms such as Mergesort.

• Incorrect choice of pivot can lead to worst-case scenarios. This problem is solved by choosing either a random index for the pivot, choosing the middle index of the partition or (especially for longer partitions) choosing the 1median of the first, middle and last element of the partition for the pivot.


Algorithm




Quicksort is a divide and conquer algorithm. Quicksort first divides a large list into two smaller sub-lists: the low elements and the high elements. Quicksort can then recursively sort the sub-lists.


The steps are:

1. Pick an element, called a pivot, from the list.

2. Reorder the list so that all elements with values less than the pivot come before the pivot, while all elements with values greater than the pivot come after it (equal values can go either way). After this partitioning, the pivot is in its final position. This is called the partition operation.

3. Recursively apply the above steps to the sub-list of elements with smaller values and separately to the sub-list of elements with greater values as shown in above figure. 

The base case of the recursion is lists of size zero or one, which never need to be sorted.
