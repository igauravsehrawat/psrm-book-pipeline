ROUND ROBIN CPU SCHEDULING

 

The idea behind developing the same may be:

  1. We initially had FCFS scheduling algorithm for CPU scheduling where the process that came first in the queue was executed first . The drawback of this was that it increases the overall waiting time for the processes . For eg:

The waiting time for P1, P2,P3 is 0 ms, 24ms and 27ms respectively provided all the processes are coming for execution at the same time. So the average waiting time will be (0+24+27)/3= 17 ms.

However, if the processes were executed in some different manner the avg. waiting time would have been much lesser. For example:

    0                               3                                6                                                                                                        30

Here the average waiting time for P1, P2, P3 is 6, 0 and 3 respectively. So the average waiting time is (6+0+3)/3=3ms. The reduction is substantial and hence one could easily see that FCFS is not a very effective method for CPU scheduling.

 

Therefore there must have been the need to create an algorithm where the waiting time gets reduced because the second case is an ideal one; but processes need not come in an ideal way. They are always processes in the sequence in which they arrive in the ready queue.

 

So one might have thought of executing processes with smaller burst time first, so that the overall waiting time reduces. But again the problem can be that the process having more priority and higher burst time might get delayed .Secondly there may be a possibility that processes with lesser burst time keep on coming and the process with higher burst time may die because of starvation.

So still the problem was not solved .So some other algorithm was needed, which could solve the problem of waiting time and not lead to starvation.

 

From the above conclusions the question was that which process should be given CPU first and for how much time. One of the solutions could be that the process having highest priority should be given CPU first and the remaining accordingly but the problem was that, this could also lead to starvation of process having less priority. So it might have been concluded that deciding which process goes first was difficult and less fruitful so it is better to have constraint on the time for which CPU will be given to a process in order to prevent starvation and reduce overall waiting time. To implement the constraint on the time for which CPU will be given to a process concept of time stamp might have brought so that no process holds the CPU for way too long, each of the process gets to have CPU in (n-1) time stamp if n is the number of processes and the average waiting time can also be made optimal.

 

So this might have been the idea behind developing round robin algorithm for CPU scheduling where the ready queue is similar to FCFS but with pre-emption involved in it. Here each process gets CPU on the basis of first come first served but for one time stamp and next time it will only get the CPU if all d processes in the ready queue already got the CPU once or there is no other process in the ready queue which needs CPU. Here the process lives the CPU in two condition either the process has finished before the time stamp and leaves the CPU itself or one time stamp is over and now CPU has to be allocated to some other process.


                                                     

K-means Algorithm:

It is a partitioning method whereby we organize the objects of a set into several data sets.

Formally, given a data set D on n objects and k number of cluster to form , k-means algorithm organizes the objects into k clusters and the objects within a cluster are similar to one another. This s the centroid based partitioning technique where the centroid represents the cluster.

So how did the algorithm come into place?

Firstly it is very difficult to analyze or conclude anything from a large data set as the data may be highly varied. So may be someone would have thought that it could be easier to interpret and conclude anything if the data had been in a smaller set , so with this he/she might have thought about breaking the data up into several parts to get a better understanding.

Then in the thought process there might have been a possibility that the person might have thought that the similar data should be organized together rather than just having different chunks of data with random data entries , so he might have thought of searching an algorithm where he could organize the data within different data chunks.

In maths we have a method with which we find the variance of a certain entity with respect to a fixed entity. So, the person might have thought of applying the mathematical concept in data mining, where with the consideration of any numerical attribute (one of the disadvantages as nominal attributes could not be considered)  one could easily categorize the different data sets.  So may be due to the high variance in the data set , the person might have thought of taking random centroids and then calculating the distances with respect to those centroids. But with these also , may be he would have tried taking other centroids and might have got better clusters. So due to this he might have decided to keep finding the new centroids for the already formed clusters and checking them with all the data entries to form better clusters. And we keep on reducing the discreetness in the clusters by repeatedly changing the fixed values with which the data is compared to.
